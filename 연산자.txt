연산자

산술 연산자
수학적 계산을 위해 사용되는 연산자
- + - * / // % **

복합 연산자
연산과 할당이 같이 이뤄짐
+= -= *= /= //= %= **=

a += b -> a = a + b

a -= b -> a = a - b

a *= b -> a = a * b

a /= b -> a = a / b

a //= b -> a = a // b

a %= b -> a = a % b

a **= b -> a = a ** b


비교 연산자
두 값을 비교하여 그 관계가 맞는지 틀리는지 True or False로 반환
< <= > >= == 1= is is not

== 연산자
* 값(데이터)이 같은지 비교하여
* 동등성
* 예를 들어, 1 == True인 경우 파이썬이 내부적으로 True를 1로 간주할 수 있으므로 True 결과가 나온다.


is 연산자
* 객체 자체가 같은지 비교
* 식별성
* 두 변수가 완전히 동일한 객체를 기리키는지, 즉 메모리 주소가 같은지 확인할 때 사용


is 대신 ==를 사용하는 이유
* is는 정체성을, ==는 가치를 비교하기 때문 -> is를 값 비교에 사용하면 안됨
* 두 연산자는 "같다"를 확인하는 목적이 근본적으로 다름


is는 언제 사용하는가?
* 주로 싱글턴 객체를 비교할 때 사용

싱글턴 객체란?
* 특정 값에 대해 파이썬 전체에서 단 하나의 객체만 생성되어 재사용되는 특별한 개체
* 여러 변수가 이 값을 가져도, 모두 미리 만들어진 하나의 객체를 함께 가리킴 -> 항상 같은 메모리 주소를 가징
* 파이썬의 대표적인 싱글턴 객체: none, true, false

*** 값이 같은지 확인하려면 == 사용
*** 두 변수가 완전히 동일한 객체를(메모리 주소를) 가리키는지 확인하려면 is 사용


논리 연산자
여러 개의 조건은 결합하거나, True/False 값을 반대로 뒤집을 때 사용
and or not

print(True and False)  # False
print(True or False)  # True
print(not True)  # False
print(not 0)  # True


단축 평가
논리 연산에서 두 번째 피연산자를 평가하지 않고 결과를 결정하는 동작
(결과가 이미 정해진다면 굳이 뒤에 있는 코드까지 확인하지 않음)

파이썬의 참과 거짓에 대한 새로운 시각
* 단축 평가를 이해하려면, 파이썬이 어떤 값을 참으로 보고 어떤 값을 거짓으로 보는지 알아야 함
* 거짓으로 취급되는 값 -> False, 숫자 0, 빈 문자열 "", 빈 리스트 [], None 등 비어있거나 없다는 느낌의 값
* 참으로 취급되는 값 -> True 그리고 거짓이 아닌 모든 값, 1, -10, "hello", [1, 2] 등 내용이 있는 값

and 연산자
*** and는 연산을 왼쪽에서 오른쪽으로 진행하다가, 처음 만나는 거짓 값을 바로 반환
*** 만약 끝까지 갔는데 모든 값이 참이면 맨 마지막 참 값을 반환

or 연산자
*** or은 연산을 왼쪽에서 오른쪽으로 진행하다가, 처음 만나는 참 값을 바로 반환
*** 만약 끝까지 갔는데 모든 값이 거짓이면 맨 마지막 거짓 값을 반환


멤버십 연산자
특정 값이 시퀀스나 다른 컬렉션 안에 포함되어 있는지 확인하는 연산자
in not in

in
* 왼쪽 피연산자가 오른쪽 피연산자의 시퀀스에 속하는지를 확인

not in
* 왼쪽 피연산자가 오른쪽 피연산자의 시퀀스에 속하지 않는지를 확인


시퀀스형 연산자
시퀀스 자료형(str, list, tuple)에 특별한 의미로 사용되는 연산자
+는 시퀀스를 연결하는 기능
*는 시퀀스를 반복하는 기능

+: 결합 연산자
*: 반복 연산자
